Overview

Here we'll begin our study of graphical user interfaces a.k.a. GUIs, in other words, where we'll use user-friendly objects to collect and communicate information to the user. The interface is built from various components. We'll begin our study by looking at the most commonly-used components (sometimes called widgets, short for window gadgets):

  • labels
  • text fields
  • buttons
  • text areas
  • checkboxes
  • radio buttons
  • drop-down lists
  • lists allowing single and multiple selection
  • panels to group components

We could separate a program into its front end, or just the user interface components, and its back end, the code behind the scene that processes what happens when the user interacts with the interface. This set of notes is concerned only with the front end. We'll look at how to set up several GUI components. In doing so, we'll separate the the definition of GUI objects from their construction, as we'll generally want to define GUI components as class-wide variables (in an applet or application class) so that that other methods can do event-handling to implement the back end.

Labels

A label is a GUI element used consisting, generally, of uneditable text to tell the user what other parts of the GUI are for. It could also include images, implemented through icons, as we saw in our introduction to graphics. We generally use labels in place of the output prompt in our old prompt-then-input pattern.

The type for a label is JLabel, and we declare one just like any other object: 

JLabel demoLabel;

The constructor you're most likely to use takes one argument: the text of the label. In general, its call looks like this: 

demoLabel = new JLabel("Label text goes here");

We also need to add the label to the GUI. If we're in a class that extends JApplet, JPanel, or JFrame, we can simply call the add method to add GUI components. Otherwise, we'd need to add to instances of them, as we did in the graphics notes when we looked at labels.

For example, we might use the following lines of code to add a label for a prompt for a user's name that might precede a text field (I'll do this as a private variable within a class): 

// Declaration goes with class members
private JLabel nameLabel;

// Code goes in init() of applet or constructor of application
nameLabel = new JLabel("Enter your name here:");
add(nameLabel);

Note that in the case of JLabels, we'll almost never need to refer to the object by name. (Next class, we'll learn how to retrieve the inputs from most GUI components, but JLabels never receive input.) Thus, we could do all of the above in one line:

add(new JLabel("Enter your name here:"));

I advocate the latter practice to keep the number of class member data to a minimum.

 

Customizations

There are some customizations you can make to a name JLabel.

The following JLabel member functions allow customizations:

  • void setToolTipText(String toolTipText) will allow you to define a tool tip, i.e. text that will appear when the user positions the mouse over the label. The argument is the text to be displayed.
  • void setText(String labelText) will allow you to change the text of a label, or initially set it if you've defined the label using the default JLabel constructor.
  • void setIcon(ImageIcon icon) will allow you to add an icon to an existing label.
  • When working with a label with an icon, setHorizontalTextPosition(int position) and setVerticalTextPosition(int position) will allow you to define the position of the text part of the label relative to the label. The position values may be the following:
    • For horizontal: SwingConstants.LEFT, SwingConstants.CENTER, SwingConstants.RIGHT
    • For vertical: SwingConstants.TOP, SwingConstants.CENTER, SwingConstants.BOTTOM

Text Fields

A text field is a GUI element where the user can type in text (or numbers) in a single line. The type for a label is JTextField, and we declare one just like any other object: 

private JTextField demoField;

The constructor you're most likely to use takes one argument: the width of the field (number of columns). In general, its call looks like this: 

demoField = new JTextField(width);

Running with the previous example, the following code creates the text field for the user to type in his/her name: 

// Declaration goes with class members
private JTextField nameField;

// Code goes in init() of applet or constructor of application
nameField = new JTextField(10);
add(nameField);

In this case and in the case of all GUI components that receive input, the JTextField must be a class member.

 

 

Password Fields and Uneditable Fields

As you may also decide to use a text field for output of information or lock the input at some point, the member method setEditable() is provided. It takes arguments of true or false.

A subclass of JTextField is JPasswordField, which operates in the same way as a regular text field, except that characters to hide the input are shown in lieu of what the user actually typed.

Text Areas

A text area is very similar to a text field, except that text fields are limited to one line of text, while text areas are multiple lines long. The type for a label is JTextArea, and we declare one like this: 

private JTextArea demoTextArea;

The constructor takes three arguments: default text for the text area, the number of columns wide the text area is, and the number of rows high the text area is. In general, its call looks like this: 

demoTextArea = new JTextArea( "default text" , # rows ,  # columns);

Related to the concept of text areas is that of scrollbars, implemented through class JScrollPane. Scrollbars must be attached to something they're scrolling, and it's a good idea to include them with text areas. By default, scrollbars appear only when needed. (See Section 14.20 of Deitel to learn how to change this if you want.) Generally, we do not need to refer to a scrollbar by name outside of constructing it, so here is how to add a scrollbar to a text area: 

add(new JScrollPane(name of text area));

Here's example code to create a text area 50 columns wide by 5 rows tall, with a scrollbar, for a user to enter comments. 

// Declaration goes with class members
private JTextArea commentField;

// Code goes in init() of applet or constructor of application
commentField = new JTextArea("Type comments here", 5, 50);
add(commentField); 
add(new JScrollPane(commentField));

Checkboxes

A checkbox is a GUI element that allows a user to turn some setting on and off, essentially with a Boolean variable behind the scenes. Note that checkboxes do not need labels; they contain labels for whatever is "being checked" with them. Checkboxes are implemented through class JCheckBox.

Here's the general form of a checkbox declaration:

private JCheckBox demoCheckbox;

The constructor you're most likely to use takes one argument: the text of the checkbox, i.e. the label. In general, its call looks like this: 

demoCheckbox = new JCheckBox("text");

Here is example code that might be added to the logon sequence we wrote above: 

// Declaration goes with class members
private JCheckBox rememberCheck;

// Code goes in init() of applet or constructor of application
rememberCheck = new JCheckBox("Remember my email address");
add(rememberCheck);

Radio Buttons

Checkboxes are square in shape, and although several of them can be placed nearby, there is no logical linkage between them. That is to say that users can check as many checkboxes as he/she wishes. This is contrast to the related radio button, which is circular in shape and which is designed such that several radio buttons are linked together. Only one radio button in a group may be selected.

Here is a visual example of the distinction:

Checkbox 1
Checkbox 2
Checkbox 3 		Radio Button 1
Radio Button 2
Radio Button 3

Radio buttons are implemented through class JRadioButton.

Here's the general form of a radio button declaration:

private JRadioButton demoRadio;

The constructor you're most likely to use takes one argument: the text of the radio button, i.e. the label for the choice. In general, its call looks like this: 

demoRadio = new JRadioButton("text");

We add radio buttons to the applet/application as before, but we also need code to link radio buttons logically. For this, we need to define a ButtonGroup object, defined as follows: 

ButtonGroup demoGroup;

demoGroup = new ButtonGroup();

Then we need to add each radio button to the group with the add method of ButtonGroup, not to be confused with the add method for adding elements to the GUI. That goes like this: 

ButtonGroupObjectName.add(JRadioButtonObjectName);

 

Here is example code that creates a group of radio buttons for choosing a color: 

// Declaration goes with class members
private JRadioButton radioBlack;
private JRadioButton radioRed;
private JRadioButton radioGreen;
private JRadioButton radioBlue;

// Code goes in init() of applet or constructor of application
ButtonGroup radioColors;                         // group to link radio buttons

radioBlack = new JRadioButton("Black", true);    // create radio buttons
radioRed = new JRadioButton("Red", false);
radioGreen = new JRadioButton("Green", false);
radioBlue = new JRadioButton("Blue", false);

add(radioBlack);                                 // add them to GUI
add(radioRed);
add(radioGreen);
add(radioBlue);

radioColors = new ButtonGroup();                 // logically link buttons
radioColors.add(radioBlack);
radioColors.add(radioRed);
radioColors.add(radioGreen);
radioColors.add(radioBlue);

Drop-Down Lists

A drop-down list is similar to a set of radio buttons in that it allows the user to choose exactly one option from several choices. Here's what a drop-down list looks like:

Drop-down lists are implemented through class JComboBox.

Here's the general form of a drop-down list declaration:

private JComboBox demoDropDown;

The constructor you're most likely to use takes one argument: an array of Strings which are the various choices. In general, its call looks like this: 

demoDropDown = new JComboBox( choices array name );

One member function worth pointing out is setMaximumRowCount(), which takes one argument that is the maximum number of rows the user sees at once.

Here's code that implements the choice of a color through a drop-down list:

// Declaration goes with class members
private static final String[] COLORS = {"Black", "Red", "Green", "Blue"};
private JComboBox colorMenu;

// Code goes in init() of applet or constructor of application
colorMenu = new JComboBox(COLORS);
colorMenu.setMaximumRowCount(3);
add(colorMenu);

Single and Multiple-Selection Lists

A list is very similar to a drop-down list, except that more than one choice is visible at a time it also allows the user to choose more than one option. Here's what a list looks like:

Lists are implemented through class JList.

Here's the general form of a drop-down list declaration:

JList demoList;

Like with drop-downs, the constructor you're most likely to use takes one argument: an array of Strings which are the various choices. In general, its call looks like this: 

demoList = new JList( choices array name );

Note that some versions of the Java compiler will require you to add a specifier that the list is a list of strings, e.g.: 

demoList = new JList<string>( choices array name );

However, we have one setting we must make: what kinds of selections the user is allowed to make. We do this by calling the modifier setSelectionMode(). There are three valid arguments:

  • ListSelectionModel.SINGLE_SELECTION, which allows the user to choose exactly one value from the list.
  • ListSelectionModel.SINGLE_INTERVAL_SELECTION, which allows the user to choose several contiguous values from the list.
  • ListSelectionModel.MULTIPLE_INTERVAL_SELECTION, which allows the user to choose several noncontiguous values from the list.

 

Here's code that implements the choice of a multiple colors through a list:

// Declaration goes with class members
private static final String[] COLORS = {"Black", "Red", "Green", "Blue"};
private JList colorList;

// Code goes in init() of applet or constructor of application
colorList = new JList(COLORS);
colorList.setSelectionMode(ListSelectionModel.MULTIPLE_INTERVAL_SELECTION);
add(colorList);

Panels

We'll talk about making GUIs neat later, but one simple way of keeping a GUI neat into organize elements into a panel. Elements in a panel stick together and do not wrap at the right margin. You might opt to keep related inputs in a panel, like related text fields with their labels or a group of checkboxes or radio buttons. Panels are implemented through class JPanel.

Here's the general form of a panel declaration and initialization:

JPanel demoPanel;

demoPanel = new JPanel();

Once we've created a panel, we add GUI components to the panel with the add method of JPanel. That goes like this: 

PanelObjectName.add(GUIComponentName);

 

Suppose we wanted to extend our radio button example so that all buttons were grouped into a panel. Here's code for that: 

// Declaration goes with class members
private JRadioButton radioBlack;
private JRadioButton radioRed;
private JRadioButton radioGreen;
private JRadioButton radioBlue;
private JPanel       colorsPanel;

// Code goes in init() of applet or constructor of application
ButtonGroup  radioColors;
radioBlack = new JRadioButton("Black", true);    // create radio buttons
radioRed = new JRadioButton("Red", false);
radioGreen = new JRadioButton("Green", false);
radioBlue = new JRadioButton("Blue", false);

colorsPanel = new JPanel();                      // create panel

colorsPanel.add(radioBlack);                     // add buttons to panel
colorsPanel.add(radioRed);
colorsPanel.add(radioGreen);
colorsPanel.add(radioBlue);

add(colorsPanel);                                // add panel to GUI

radioColors = new ButtonGroup();                 // logically link buttons
radioColors.add(radioBlack);
radioColors.add(radioRed);
radioColors.add(radioGreen);
radioColors.add(radioBlue);
