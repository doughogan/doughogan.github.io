Colors & The RGB Model

Remember that everything in computers is ultimately represented as binary numbers. We can specify other colors using the RGB model, which specifies a color with a red value, a green value, and a blue value (each called channels). For example, the red value is how much red we want in our color. In different graphics system, the individual color values might be represented in different ways. One very common way is to use 8 bits to each color value. This gives us 256 (28) choices for each of the R, G, and B values, where choices range from 0 to 255.

So, we could specify a color as follows:

Component Minimum Value Maximum Value
Red Value 0 (no red) 255 (all red)
Green Value 0 (no green) 255 (all green)
Blue Value 0 (no blue) 255 (all blue)

So, a color is represented as an ordered triple.

Here are some examples:

  • Black is (0, 0, 0), i.e. the absence of all light, so 0 for all channels
  • White is (255, 255, 255), i.e. the presence of all light, so all channels are fully "on."
  • We could have two different channels fully "on," so, for example, (255, 0, 255) is magenta
  • We could use values between 0 and 255, where larger means brighter, so (50, 0, 0) is a dark red and (200, 0, 0) is a brighter red, while (0, 80, 0) and (0, 220, 0) are two different shades of green.
  • We could combine the above ideas. If we were to change (0, 220, 0) to (0, 220, 220) or (0, 220, 20), what would be different?

Observe that 256 is also 162, which means a component of color can also be represented as 2 hexadecimal digits. Going with this, FF == 255. In other graphics systems, a triple is represented by 6 hex digits, where the first two are the red value, the second two are the green value, and the last two are the blue value. Thus, FF0000 is red, 00FF00 is green, and 0000FF is blue. We could combine colors: FF00FF is magenta . (Be careful, though, it's more involved than this.)

In HTML, we can specify colors using hexadecimal numbers.
