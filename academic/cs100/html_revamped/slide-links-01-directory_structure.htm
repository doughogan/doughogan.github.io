Directory structure

In a WYSIWYG editor, you can use the GUI to find the paths of links and images. In HTML, you need to refer to the exact path of the file.

HTML is designed to work with web servers. Web servers are generally based on Unix operating systems. Hence, we use forward slashes, /, to denote changes in directories. (Microsoft stole the idea but changed it ever so slightly to backslashes, \, so what you see in Windows is slightly different.)

File paths are always relative to the current directory. The notation .. (two dots) is used to denote "one level up."

Some general forms and examples:
