Vertical Navigation Bars and Columns

Many pages on the web are divided into columns of content. Often you'll find a vertical navigation bar on the left side of the page and content on the right. How is this implemented?

It turns out you don't need to learn anything new here: the implementation uses nothing more than tables. We might use the following sorts of settings

  • Don't set a border to hide the table behind the scenes (it's only an implementation detail)
  • Create a table with 1 row and as many columns of content as you'd like. Usually this is two columns.
  • Use the width attribute of <td> to control how much of your page is reserved for content and how much is reserved for navigation (or to keep widths of columns of text equal).
  • Use cell padding and/or cell spacing to keep contents from running together.

It's wise to create the outer structure of the page first, then put the contents of the page in the proper cells.

Example 1: A Vertical Navbar

HTML As displayed in browser 
<table cellpadding="4">
	<tr>
		<td width="30%" valign="top">
			<!-- code for navbar -->
			Link 1<br>

Frames

Another way to create navigation quickly is to use frames, in other words, divide the browser window up into sections called frames, each holding a different web page. Here are the steps of this process:

  1. Create a frameset, the HTML page that divides the browser window into frames, gives each frame a name, and tells how much space to allocate for it. Standard convention also dictates that this page also has a provision for browsers which don't support frames.
  2. Create the HTML pages for each frame.
  3. Target the links in each frame so that they open in the correct frames.

Frameset

The frameset is the page that defines the frame structure. It also sets the page title that is actually seen in the browser.

In this page, you must create the outer structure with a <frameset> tag. This tag takes these attributes:

  • Either a rows or a columns attribute, whose value is a comma separated list telling the percentage of the screen (or size in pixels) to allocate for each frame.
  • Optionally, take a frameborder attribute, whose value is the width of the frame divisions in pixels.

Here's an example:

<frameset cols="30%, 70%" frameborder="0">

   <!-- code for frames goes here -->

</frameset>

Within this structure, we must define each frame. We do this with the <frame> tag, which has two required attributes:

  • src, whose value is the path or URL of the page to be displayed in the frame
  • name, whose value is the name of the frame. (While it doesn't seem important yet, it's incredibly important in making the links work.)

We need a <frame> tag for each frame in the frameset. Here's the last example with the frames filled in:

<frameset cols="30%, 70%" frameborder="0">

  <frame src="navframe.htm" name="nav">
   <frame src="main.htm" name="contents">

</frameset>

Finally, we need to keep in mind that, historically, not all browsers supported frames. While it's rare to encounter this these days, it's still wise to put a good error message there. Ten years ago, it would have been recommended that you make a completely alternate site for this case, but now an error message is generally sufficient. We put this warning (or alternate content) within the <noframes>...</noframes> tag pair, which should appear after the <noframes>...</noframes> tag pair.

Here's an example:

<noframes>This page uses frames, which your browser does not support. Please upgrade your browser to...</noframes>

 

Pages for Frames

This part is nothing new. We just write our HTML like we always would. We do need to keep in mind that frames don't get as much browser real estate and plan accordingly.

 

Targeting Links

When you put links into pages that are to be displayed within frames, you need to tell the browser what frame to open the link in. This is done by adding a target attribute to your link tag. The value of this attribute is the name of the frame the page should open in.

Here's the general form:

<a href="path_or_url" target="name_of_frame_in_current_frameset">What user clicks</a>

Two special values of target to be aware of when working with frames:

  • target="_self" sends links to the current frame.
  • target="_parent" replaces the current frameset completely.

Enrichment Note: More Efficient Layouts with PHP

By the way, in practice, web developers rarely copy and paste navigation bars from page to when using tables for layouts. There are more appropriate tools for this, particularly PHP. Here's the general idea of how common navigation might be done using PHP:

  1. Create the navigation bar (or other banners or common content) in its own PHP file (could likely be all HTML).
  2. Create the structure of the page in a separate PHP file, again using the HTML we know and love. Use tables as needed.
  3. In the file users will see, go to where you want your common content and mark off a section as PHP. Call PHP's include function to import the first file here. (Example: <? include 'header.php' ; ?>)
  4. Enter the content of your page as usual.

This allows you to get the write-once-use-many benefits of navigation bars and common content that you'd have with frames but with the benefits of having your entire page in a single file on the user's end.

Spawning New Windows

You can make a link open in a new window by simply adding a target attribute to any link. This also uses the target attribute, but you can do it any page, not just those within frames. Here's the general form:

<a href="path_or_url" target="_blank">What user clicks</a>

Anchors

An anchor is a point within an HTML file. They are useful if we wish to make a link to specific part of a page, rather than the whole page.

Anchors require a form of the <a> tag we saw earlier to define them:

<a name="name_of_anchor"></a>

Links can be made to anchors within the current page by using the standard link tag with a path beginning with a #:

<a href="#name_of_anchor"></a>

Links can be made to anchors in other pages by appending a a # and an anchor name to the the path:

<a href="filename.html#name_of_anchor"></a>

Lots of example uses of anchors can be found throughout the class notes.
