Standalone Programs (vs. Interactive Mode)

Last week, all of our uses of Python were in interactive mode. That mode is very practical for quick calculations, but Python is a powerful programming language in its own right and we can use it to create programs that stand on their own. We'll look at programs that exist in one file in this lecture.

Like with HTML, use a plain text editor such as Text Edit or Notepad to write your code. Save your code with the extension py.

As Python is an interpreted language, we need an interpreter to run Python programs. There's a command you enter at the command line to run your programs. To run programs, use the command python, followed by the name of the program.

Hello World!

It's standard practice that the first program people write in any language is the "Hello World" program, i.e. a program which simply displays the text "Hello, World!" so we'll start with that. In Python, it's quite simple. The program is just one line:

print("Hello, World!")

This code should be saved in a file called hello.py. To run it, we'd type this command:

python hello.py

And the computer would display this:

Hello, World!

Getting Input from Users

Last week, we talked about setting the values of variables with assignment statements. Certainly, we would want to have the ability for programs to interact with users. To get information from a user in a console program, we follow what is called the prompt-then-input pattern, in other words, we first display a message to the user saying what we want (the prompt) and then wait for the user to do input.

We must store our input somewhere, and Python's way of doing this looks like an assignment statement.

The general form fits in one line of code and looks like this:

receivingVariableName = eval(input("prompt goes here"))

For example, the following would ask the user for a number and store it in num1: 

num1 = eval(input("Enter a number: "))

As a second example, the following would ask the user for a length and store it in length: 

length = eval(input("Enter the length in inches: "))

Observe the space after the colons in my prompts; that's intentional. Otherwise, what the user types would be right beside the prompt. Also, note that it's important to be clear in your prompts. In my second example, I'm careful to include the units I'm expecting (and assuming in later calculations).

Short, Complete Programs and IPO

Now we have the skills to write a complete program. Programs can take many forms, but most involve these three parts:

  • Input: Collecting information from a user
  • Process: Doing some calculation on the input collected or transforming it in some way
  • Output: Reporting results back to the user

Because of these three phases, many programs are written to follow the IPO model, where the name comes from the first letters of the three phases given. (Strictly speaking, the IPO model requires that each of these phases be distinct. That won't be possible with our limited toolkit in some later examples today though.)

Here's a first example of a complete program: 

num1 = eval(input("Enter a number: "))
num2 = eval(input("Enter another number: "))

sum = num1 + num2

print("The sum of your numbers: ", sum)

When it's run, it might look like this (where what the user has typed is in color): 

Enter a number: 2
Enter another number: 4
The sum of your numbers:  6

Documenting Code

We can add text called comments to our code that is not meant for the computer, but rather meant for ourselves and other programmers. Any text in Python that you type after typing a # symbol on a line is considered a comment. It is ignored by the computer, but can help you in organizing your code, can help other programmers understand your code, and can help you with maintaining programs at times other than after they're written.

Here's the previous program with some organizing comments added: 

# Input: Collect two numbers from the user
num1 = eval(input("Enter a number: "))
num2 = eval(input("Enter another number: "))

# Process: Add the two inputs
sum = num1 + num2

# Output: Report the sum 
print("The sum of your numbers: ", sum)

I'll include comments in some later examples.

Here in CS 100, you don't need to write any comments to satisfy requirements (but, by all means, add them as it helps you). However, if you choose to learn more programming, it is very important to get in the habit of documenting your code with good comments while you write it.

Loop Applications

We talked briefly about loops last week. Recall that the for loop lets us do the same thing a predetermined number of times, and is controlled by a counter (or loop control variable). Recall the general form:

for i in range(start_value, one_after_stop_value, step_value):
   statement(s) to be repeated

Everything last time was just concerned with simple counting. We'll now apply the for loop to a few standard important places loops are used. As you may have gathered from our conceptual activities, processing lists of information is something we do often in CS. In these examples, we'll collect one input at a time and do something with it.

 

Summing Values

There are plenty of cases where we want to add up values.

To do so, we need two variables:

  • one to hold the current number being input, which I'll call cur
  • one to hold the sum we're accumulating, which I'll call sum

We'll simply read in a number and replace the value in cur after each input.

Before we start doing anything, we need to give sum an initial value, called initializing the variable.

Question: If we have added up no values, what is the sum of those values?

 

We need to know how many inputs in advance. Let's say we'll read in 10 values here. We could start the loop as follows: 

for i in range(1, 11, 1)

Note that it would be equivalent to do the following, as it also counts 10 times:

for i in range(0, 10, 1)

 

Once we've read in an input, we add it to the existing sum. The complete code is as follows:

sum = 0                                   # initialize sum

for i in range(1, 11, 1):                 # collect and sum inputs
   cur = eval(input("Enter a value: "))
   sum = sum + cur

print("Total of inputs is: ", sum)

Question: Say we want to read in 100 numbers instead of just 10? What do we need to change?

 

Counting Things

Let's say we want to read in 100 numbers and see how many are even.

Question: Before we've looked at any inputs to see if they're even, how many even numbers have we seen?

 

Question: When we divide an even number by 2, what's the remainder? How do we compute a remainder in Python?

 

There's one other new issue. We've learned about if statements before. Nothing says we can't use them in the body of loops. In fact, doing so is extremely common and powerful.

Putting all of this together, we get the following:

evenCount = 0                                   # initially, no evens

for i in range(0, 10, 1):                       # collect and test inputs
   cur = eval(input("Enter a value: "))
   if cur%2 == 0:                               # encountered an even value...
      evenCount = evenCount + 1                 # ...so tally it

print("Number of even inputs: ", evenCount)

 

 

Finding Extreme Values

Recall when we discussed the idea of finding a maximum early in the term. Turning that algorithm into code is very similar to what we did above. Here's the code:

max = eval(input("Enter a value: "))       # get first value, initially the max

for i in range(0, 10, 1):                  # collect and test inputs
   cur = eval(input("Enter a value: "))
   if cur > max:                           # check if new input beats the max...
      max = cur                            # ...and make it the new max if it does
   print("Maximum so far: ", max)

print("Maximum found: ", max)

Other Variable Types: Booleans and Strings

It is worth taking a moment to discuss data types. There are three basic types of things we could store in a variable:

  • numbers: As per every example we've seen so far. Some applications will distinguish between whole numbers and those with decimals.
  • text: Letters or several letters or even digits, referred to as strings.
  • Boolean: Values of true or false, literally using keywords True and False in Python

In some languages, the notion of data types comes up in every program you write and you must say the type of variables before first use them. Such languages are called strongly-typed. This is not the case in Python. The language infers the types from how variables are used. In some sense, this means you can do more with less code, but on the other hand, it puts more responsibity on the programmer.

While Loops

We know about for loops, used when we know how many times we want to do something in advance. There's a second kind of loop in Python, the while loop. While you can do anything with a while loop you can do with a for loop, it is best used for the cases when you do not know how many times you want the loop to do something in advance.

Here's the general form:

while loop test:
   statement(s) to be repeated

The loop test is a Boolean expression, as we'd use in an if statement. In fact, the behavior of a while loop is similar to an if statement in that the body runs when the condition is true. The difference with a while loop is that the condition is checked again after the body, and if it remains true, the body runs again.

To use a while loop, we nearly always must:

  • Initialize something before the loop.
  • Have code inside the loop that makes it possible the loop test might fail, in other words, bring the loop closer to stopping.

Here's an example from the initial discussion of for loops:

for i in range(1, 11, 1):
   print(i)

While that truly is a job for a for loop, here's a while loop that does the same thing:

i = 1

while i < 11:
   print(i)
   i = i + 1

 

Here is code that does the same thing as the counting example from earlier:

sum = 0                                   
i = 1

while i < 11:               
   cur = eval(input("Enter a value: "))
   sum = sum + cur
   i = i + 1

print("Total of inputs is: ", sum)

But, I still haven't really given you a reason to care about this new kind of loop! Let's do that now. What if, instead of stopping after 10 inputs, we assume the user only cares about positive inputs? So, we could let the user enter -1 as a way of stopping. Here's how the code changes:

sum = 0
cur = 1

while cur != -1:               
   cur = eval(input("Enter a value or -1 to quit: "))
   sum = sum + cur

print("Total of inputs is: ", sum)

Note that because we need to use cur in the loop test, we need to initialize it to something before the loop to make the loop test true. (Admittedly, this is a bit unnatural, but there's no way to get around it in Python without repeating code and is a limitation of the language.)

More Loop Applications

Let's look at two more applications of loops.

Mystery Code

Consider this block of code:

key = 35

position = 1
found = False

while not found:
   cur = eval(input("Enter a value: "))
   if cur == key:
      found = True
   else:
      position = position + 1

print(key, " is the ", position, "th value")

What does it do? (Hint: It's an algorithm we discussed conceptually earlier in the term...)

 

 

Something with a Mystery, More Literally

As a last example, let's look at an example which leverages the while loop to implement a guessing game (and, as a bonus, shows an elif option for decision making in a case of three-way branching):

mysteryNumber = 65                                               # number to guess

isGameOver = False                                               # flag that's false while game
                                                                 # is underway and flips when done

while not isGameOver:                                            # let user guess values
   cur = eval(input("Guess a number: "))                         # get a new guess

   if cur == mysteryNumber:                                      # guess is right
      print("Congratulations, you guessed the mystery number!")
      isGameOver = True
   elif cur > mysteryNumber:                                     # guess is too big
      print("Sorry, too high! Try again...")
   else:                                                         # guess is too small
      print("Sorry, too low! Try again...")
